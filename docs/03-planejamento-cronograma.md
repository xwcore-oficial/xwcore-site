# Planejamento e Cronograma

## Objetivo Geral

Definir a base institucional da XWCore e organizar os projetos em andamento para avançar com desenvolvimento, documentação, presença digital e publicação inicial.

## Fase 1 - Fundação Institucional

Período sugerido: Semana 1

Entregas:

- Conceito institucional definido.
- Identidade visual inicial.
- Logo vetorial.
- Página institucional minimalista.
- Estrutura documental no repositório.

Critério de conclusão:

- Site estático navegável.
- Documentação inicial revisada.
- Domínio principal escolhido para publicação.

## Fase 2 - Organização Técnica dos Produtos

Período sugerido: Semanas 2 e 3

Entregas:

- Repositórios definidos por produto ou monorepo organizado.
- README individual por projeto.
- Definição de status, escopo, dependências e próximos marcos.
- Roadmap técnico com prioridades.

Critério de conclusão:

- Cada produto com objetivo, público, arquitetura inicial e backlog mínimo.

## Fase 3 - Provas Funcionais

Período sugerido: Semanas 4 a 8

Entregas:

- Protótipo funcional do Interfone IP Wi-Fi.
- Protótipo funcional do Módulo RF Wi-Fi MQTT.
- Versão inicial do MonitorNFE.
- Versão inicial do DASN Simples Nacional.
- Versão inicial do XW-Copy para cadastro e execução de rotinas de backup.

Critério de conclusão:

- Demonstração interna com instalação, configuração e fluxo principal documentados.

## Fase 4 - Publicação Inicial

Período sugerido: Semanas 9 a 12

Entregas:

- Página institucional publicada.
- Página ou repositório público para documentação.
- Política básica de suporte e contato.
- Primeiras versões de documentação de usuário.

Critério de conclusão:

- Domínios apontando para a presença institucional.
- Repositório GitHub configurado.
- Canais de contato definidos.

## Prioridade Recomendada

1. Publicar a base institucional e o repositório.
2. Consolidar o Interfone IP e o Módulo RF como linha de automação.
3. Consolidar MonitorNFE e DASN como linha de ferramentas fiscais.
4. Consolidar XW-Copy como ferramenta de proteção e organização de dados.
5. Separar comunicação comercial por linhas de produto quando houver maturidade.
